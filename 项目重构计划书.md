## 一、当前项目问题分析

### 1. 回滚机制累赘性问题
**现状分析**:
- 当前项目在`repair_engine.py`中实现了完整的备份/恢复机制
- 每次修复操作都会创建`.res`备份文件
- 在`app.py`中有"恢复"和"删除备份"按钮
- 工具菜单中有"恢复上一次操作"和"删除备份文件"选项

**问题**:
- 备份文件占用额外磁盘空间
- 操作流程复杂，用户需要手动管理备份文件
- 当输出目录与源目录不同时，备份文件位置混乱
- 用户反馈：在已有输出目录的情况下，不需要备份文件，因为源文件并未损坏

### 2. 文件创建时间属性破坏问题
**现状分析**:
- `repair_engine.py`中的`repair_with_backup()`方法在修复后缀时会恢复原始文件时间信息
- `file_processor.py`中的`fix_file_time()`方法只修复修改时间，不修复创建时间
- 时间修复时会修改文件的创建时间、修改时间

**问题**:
- 某些操作可能破坏文件的原始创建时间属性
- 用户要求：操作完成后应完全保留原始文件信息，不能有任何破坏

### 3. 时间修复选项缺乏灵活性
**现状分析**:
- `RepairDialog`中只有两个复选框："修复错误后缀"和"修复时间信息"
- 没有提供时间来源选择（时间戳、修改时间、创建时间等）
- 时间修复逻辑固定：优先使用元数据时间，其次使用文件修改时间

**问题**:
- 用户无法根据需求灵活选择时间来源
- 不同场景需要不同的时间修复策略

### 4. 修复按钮功能过于集中
**现状分析**:
- `app.py`中只有一个"修复图片"按钮，打开`RepairDialog`
- `RepairDialog`包含了后缀修复、时间修复、重命名等所有功能
- 功能过于集中，用户操作不够灵活

**问题**:
- 用户可能只需要执行部分操作（如只修复后缀，不修复时间）
- 功能集中导致操作流程复杂

## 二、重构计划

### 1. 彻底删除回滚机制

#### 1.1 修改目标
- 删除所有备份/恢复相关功能
- 简化操作流程，提高用户体验
- 在操作前提醒用户备份数据

#### 1.2 具体修改

**文件: `repair_engine.py`**
1. 删除以下方法:
   - `get_backup_path()`
   - `has_backup()`
   - `create_backup()`
   - `restore_from_backup()`
   - `delete_backup()`
   - `delete_backups()`
   - `batch_restore()`
2. 修改`repair_with_backup()`方法:
   - 删除创建备份的步骤（Step 1）
   - 删除备份相关的返回字段
   - 重命名方法为`repair()`（去掉`_with_backup`后缀）
3. 修改`batch_repair()`方法:
   - 调用修改后的`repair()`方法
   - 删除备份相关的返回字段
4. 删除`BACKUP_SUFFIX`常量
5. 添加操作前备份提醒功能

**文件: `app.py`**
1. 删除以下方法:
   - `_restore_from_backup()`
   - `_delete_backups()`
2. 修改工具栏:
   - 删除"恢复"按钮
   - 删除"删除备份"按钮
3. 修改工具菜单:
   - 删除"恢复上一次操作"选项
   - 删除"删除备份文件"选项
4. 修改`_repair_with_dialog()`方法:
   - 调用修改后的`repair()`方法
5. 添加操作前备份提醒对话框

**文件: `batch_dialog.py`**
1. 修改`RepairDialog`:
   - 删除"备份机制"说明框
   - 添加操作前备份提醒说明

**文件: `single_view.py`**
1. 修改`_restore()`方法:
   - 删除备份恢复功能
   - 改为提示用户手动恢复
2. 修改`_full_repair()`方法:
   - 调用修改后的`repair()`方法

#### 1.3 替代方案
在执行操作前显示提醒对话框：
```
"重要提醒：此操作将修改您的文件。
建议您在操作前备份重要数据。
确定要继续吗？"
```

### 2. 修复文件创建时间属性破坏问题

#### 2.1 修改目标
- 确保所有操作都不破坏文件的原始属性
- 操作完成后完全保留原始文件信息
- 特别保护文件的创建时间属性

#### 2.2 具体修改

**文件: `repair_engine.py`**
1. 修改`repair()`方法（原`repair_with_backup()`）:
   - 在操作前记录所有文件属性（创建时间、修改时间、访问时间）
   - 操作完成后恢复所有原始属性
   - 确保创建时间不被修改（除非用户明确选择修改）
2. 修改`fix_extension()`方法:
   - 操作前记录原始文件属性
   - 操作完成后恢复所有属性
3. 添加文件属性保护机制:
   - 创建`FileAttributeProtector`类
   - 提供`save_attributes()`和`restore_attributes()`方法

**文件: `file_processor.py`**
1. 修改`fix_file_time()`方法:
   - 添加`preserve_created_time`参数（默认True）
   - 当`preserve_created_time=True`时，不修改创建时间
   - 修改`batch_fix_time()`方法，添加相同参数

**文件: `helpers.py`**
1. 修改`set_file_times()`函数:
   - 添加`preserve_created`参数（默认True）
   - 当`preserve_created=True`时，只修改修改时间，不修改创建时间

#### 2.3 保护策略
1. **默认保护**: 所有操作默认不修改创建时间
2. **明确选择**: 用户可以选择是否修改创建时间
3. **属性恢复**: 操作完成后恢复所有原始属性
4. **日志记录**: 记录属性变化，便于问题排查

### 3. 时间修复选项用户可选择

#### 3.1 修改目标
- 提供灵活的时间来源选择
- 用户可以根据需求选择不同的时间修复策略
- 保留自动选择作为默认选项

#### 3.2 时间来源选项
1. **自动选择**（默认）:
   - 优先使用元数据中的拍摄时间
   - 其次使用文件名中的时间戳
   - 最后使用文件修改时间
2. **使用元数据时间**:
   - 只使用EXIF/XMP中的拍摄时间
   - 如果没有元数据时间，则跳过修复
3. **使用文件修改时间**:
   - 使用文件的最后修改时间
   - 忽略元数据时间
4. **使用文件创建时间**:
   - 使用文件的创建时间
   - 忽略其他时间来源
5. **使用时间戳**:
   - 使用用户指定的时间戳
   - 需要提供时间戳输入框

#### 3.3 具体修改

**文件: `batch_dialog.py`**
1. 修改`RepairDialog`:
   - 在"修复时间信息"复选框下方添加时间来源选择
   - 添加单选按钮组：
     - 自动选择（默认）
     - 使用元数据时间
     - 使用文件修改时间
     - 使用文件创建时间
     - 使用时间戳（添加时间戳输入框）
   - 当"修复时间信息"未选中时，禁用时间来源选择

**文件: `repair_engine.py`**
1. 修改`repair()`方法:
   - 添加`time_source`参数
   - 根据`time_source`参数选择时间来源
   - 实现不同时间来源的逻辑
2. 修改`extract_time_info()`方法:
   - 添加`time_source`参数
   - 根据参数返回指定来源的时间

**文件: `file_processor.py`**
1. 修改`fix_file_time()`方法:
   - 添加`time_source`参数
   - 支持不同时间来源

### 4. 修复按钮拆分

#### 4.1 修改目标
- 将"修复图片"按钮拆分为多个专用按钮
- 每个按钮执行特定功能
- 提供更灵活的操作选择

#### 4.2 按钮拆分方案
1. **修复后缀**:
   - 功能：只修复文件后缀错误
   - 不修改时间信息，不重命名
   - 适用于：只需要修复后缀的情况

2. **修复时间**:
   - 功能：只修复文件时间信息
   - 不修复后缀，不重命名
   - 适用于：只需要修复时间的情况

3. **重命名**:
   - 功能：只重命名文件
   - 不修复后缀，不修复时间
   - 适用于：只需要重命名的情况

4. **完整修复**:
   - 功能：修复后缀 + 修复时间 + 重命名
   - 相当于原来的"修复图片"功能
   - 适用于：需要完整修复的情况

#### 4.3 具体修改

**文件: `app.py`**
1. 修改工具栏:
   - 将"修复图片"按钮替换为下拉菜单或按钮组
   - 添加以下按钮：
     - "修复后缀"
     - "修复时间"
     - "重命名"
     - "完整修复"
2. 修改工具菜单:
   - 将"修复图片（带备份）"替换为：
     - "修复后缀"
     - "修复时间"
     - "重命名"
     - "完整修复"
3. 添加新方法:
   - `_repair_extension_only()`: 只修复后缀
   - `_repair_time_only()`: 只修复时间
   - `_rename_only()`: 只重命名
   - `_full_repair()`: 完整修复（保留现有方法）

**文件: `batch_dialog.py`**
1. 创建新的对话框类:
   - `ExtensionRepairDialog`: 后缀修复对话框
   - `TimeRepairDialog`: 时间修复对话框
   - `RenameOnlyDialog`: 重命名对话框
2. 修改`RepairDialog`:
   - 重命名为`FullRepairDialog`
   - 保留所有功能

#### 4.4 用户界面设计
工具栏按钮布局：

3. **性能风险**:
   - 文件属性保护可能增加操作时间
   - 需要优化性能
