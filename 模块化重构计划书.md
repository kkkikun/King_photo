## 1. 项目现状分析

### 1.1 当前架构

项目采用三层架构：
- **UI层**: tkinter + ttkbootstrap 界面
- **Core层**: 核心业务逻辑（格式检测、元数据读写、修复引擎）
- **Utils层**: 工具函数（常量、配置、日志、ExifTool封装）

### 1.2 存在的问题

| 问题类型 | 具体表现 | 影响程度 |
|----------|----------|----------|
| **接口暴露不足** | 模块功能分散，缺乏统一API入口 | 高 |
| **模块间耦合** | Core层内部模块直接依赖，缺乏抽象接口 | 中 |
| **配置管理分散** | 配置管理器、常量定义、日志配置各自独立 | 中 |
| **插件机制缺失** | 添加新格式需要修改多个文件，无法热插拔 | 高 |
| **错误处理不统一** | 各模块错误处理方式不一致 | 中 |
| **测试覆盖不全** | 缺乏集成测试和接口测试 | 中 |

### 1.3 现有接口分析

**格式检测模块** (`format_detector.py`):
- 静态方法，无状态
- 直接访问常量 `FILE_SIGNATURES`、`SUPPORTED_FORMATS`
- 缺乏扩展接口

**元数据读取模块** (`metadata_reader.py`):
- 静态方法，依赖多个内部模块
- 直接导入 `FormatDetector`、`ExifHandler`、`XmpHandler`
- 缺乏统一的读取策略

**元数据写入模块** (`metadata_writer.py`):
- 复杂的回退机制（piexif → exiftool）
- 缺乏写入策略接口
- 错误处理不统一

**ExifTool封装** (`exiftool_wrapper.py`):
- 单例模式，但接口不够清晰
- 缺乏插件式扩展

## 2. 重构目标

### 2.1 核心目标

1. **完全模块化**: 每个功能模块独立，可单独使用
2. **接口暴露**: 提供清晰的公共API，方便外部调用
3. **插件机制**: 支持动态添加新格式和功能
4. **配置统一**: 统一的配置管理接口
5. **错误处理**: 统一的错误处理和报告机制
6. **易于扩展**: 新功能只需实现接口，无需修改核心代码

### 2.2 具体目标

| 目标 | 描述 | 优先级 |
|------|------|--------|
| **API统一** | 创建统一的API入口点 | 高 |
| **接口定义** | 定义清晰的抽象接口 | 高 |
| **插件系统** | 实现格式插件和功能插件 | 高 |
| **配置中心** | 统一配置管理 | 中 |
| **错误体系** | 统一错误处理 | 中 |
| **文档完善** | 完善API文档和使用示例 | 低 |

## 4. 接口暴露方案

### 4.1 接口设计原则

1. **单一职责**: 每个接口只做一件事
2. **依赖倒置**: 依赖抽象而非具体实现
3. **开闭原则**: 对扩展开放，对修改关闭
4. **接口隔离**: 接口小而专一

### 4.2 核心接口定义

#### 4.2.1 格式检测接口

```python
from abc import ABC, abstractmethod
from typing import Optional, Tuple, Dict, Any

class IFormatDetector(ABC):
    """格式检测接口"""
    
    @abstractmethod
    def detect_by_header(self, filepath: str) -> Optional[str]:
        """通过文件头检测格式"""
        pass
    
    @abstractmethod
    def detect_by_extension(self, filepath: str) -> Optional[str]:
        """通过扩展名检测格式"""
        pass
    
    @abstractmethod
    def get_real_format(self, filepath: str) -> Tuple[Optional[str], bool]:
        """获取真实格式"""
        pass
    
    @abstractmethod
    def is_truly_image(self, filepath: str) -> Tuple[bool, str]:
        """检查是否为真正的图片"""
        pass
    
    @abstractmethod
    def get_format_info(self, filepath: str) -> Dict[str, Any]:
        """获取格式详细信息"""
        pass
    
    @abstractmethod
    def register_format(self, format_name: str, format_config: Dict) -> None:
        """注册新格式（插件机制）"""
        pass
```

## 5. 插件机制设计

### 5.1 插件类型

| 插件类型 | 用途 | 接口 | 加载方式 |
|----------|------|------|----------|
| **格式插件** | 添加新图片格式支持 | `IFormatPlugin` | 文件扫描 |
| **功能插件** | 添加新功能（如批量处理） | `IFunctionPlugin` | 配置注册 |
| **扩展插件** | 扩展现有功能 | `IExtensionPlugin` | 动态加载 |

### 5.2 插件接口

```python
class IFormatPlugin(ABC):
    """格式插件接口"""
    
    @property
    @abstractmethod
    def format_name(self) -> str:
        """格式名称"""
        pass
    
    @property
    @abstractmethod
    def extensions(self) -> list:
        """支持的扩展名列表"""
        pass
    
    @property
    @abstractmethod
    def magic_numbers(self) -> list:
        """文件头魔数"""
        pass
    
    @abstractmethod
    def read_metadata(self, filepath: str) -> Dict[str, Any]:
        """读取元数据"""
        pass
    
    @abstractmethod
    def write_metadata(self, filepath: str, metadata: Dict[str, Any]) -> bool:
        """写入元数据"""
        pass
    
    @abstractmethod
    def get_datetime(self, filepath: str) -> Optional[datetime]:
        """获取拍摄时间"""
        pass
    
    def is_available(self) -> bool:
        """检查插件是否可用"""
        return True
```

class KingPhotoError(Exception):
    """King_photo 基础错误类"""
    
    def __init__(self, message: str, error_code: str = None, 
                 details: Dict[str, Any] = None):
        super().__init__(message)
        self.error_code = error_code
        self.details = details or {}
        self.timestamp = datetime.now()
    
    def to_dict(self) -> Dict[str, Any]:

        return {
            "error_type": self.__class__.__name__,
            "message": str(self),
            "error_code": self.error_code,
            "details": self.details,
            "timestamp": self.timestamp.isoformat()
        }
